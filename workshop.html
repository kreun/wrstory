챗봇 수업에서는 이론이 아닌 챗봇 제작을 진행합니다.
수업 전에 만들고자 하는 챗봇의 시나리오를 작성해 주세요.

어떤 수업을 들어야 할까?
UI 기능만을 사용해서, 정보 전달 위주로 챗봇을 구성하는 경우 : 카카오톡 오픈빌더
사용자의 발화(말)가 아닌, 사용자의 선택에 따라서 정보를 응답하는 형태이며, NLP를 사용하지 않는 단순한 형태의 챗봇입니다. 카카오의 오픈빌더를 통해서 쉽게 UI 를 구성하고, 흐름을 제어하는 방법을 학습합니다.

데이터를 이용하는 챗봇 만들기
외부 API를 통해서 실시간 데이터를 적용하거나, 데이터베이스를 통한 개인별 데이터를 사용할 수 있도록 학습합니다. 날씨 정보 가져오기, 오픈 API 사용하기, 서버와 연동해서 데이터 받기 등이 있을 수 있고, 오픈빌더를 통해서 입력받은 사용자의 데이터를 구글시트나 데이터베이스에 입력할 수도 있습니다.

사용자의 발화에 따라 정확한 답변을 할 수 있는 챗봇을 구성
일반적으로 말하는 AI 가 적용된 챗봇을 이야기 하며, 구글이 서비스 하고 있는 다이얼로그플로우를 통해서 빠르고 쉽게 대화형 챗봇을 제작할 수 있습니다.
강의에서 대화형 챗봇의 제작은 다이얼로그플로우를 기반으로 할 수 있습니다.


챗봇 수업 전에
구글에서 발표한 대화 디자인에 대해서 학습하는 것을 추천 합니다.
open_in_browserGoogle conversation-design
선택형 챗봇을 제작한다고 해도 마찬가지로, 챗봇의 UX는 다르므로, 전체적인 흐름을 파악하고, 어떤식으로 구성 할 지를 파악해야 합니다. 단순히 웹의 문서를 챗봇으로 구성하려고 한다면, 실패하는 프로젝트가 될 수 있습니다.
현재 서비스 중인 챗봇을 찾아서 장단점에 대해서 정리해 봅니다.
